# Phase 2 report — Bit-exact fixed-point golden model

Built `core/fxp`: `Fx`/`FxFormat` value types (FX-1), bit-exact operator functions
mirroring nkMatlib RTL (FX-2) — including a faithful non-restoring divider loop
(exact divide-by-zero behavior), the Meessen bit-serial square root, floor-direction
`snorm`, and renorm-then-wrap vector accumulation for sumsqr/matmul/crossp — plus a
DAG evaluator returning every intermediate signal keyed by node ID (FX-3), a float64
reference evaluator with max-abs/RMS/SQNR error statistics (FX-4), and hypothesis
property suites of 200–400 cases per operator (FX-5: wrap consistency, sdiv
truncation toward zero, usqrt == isqrt(raw·2^S) and monotonicity, snorm round-trip).
`docs/fxp_semantics.md` cites the defining SV file/line for every decision;
the reviewed checklist is `docs/fxp_semantics_checklist.md`.

- Tests: 127 passed. Coverage: core 94 % (gate: 90 %). ruff + mypy --strict(core): clean.
- Notable finding: the README's claim that snorm "rounds towards 0" is inaccurate
  for negative values — the RTL bit-slice floors (toward −∞); the RTL behavior wins (FX-2).
- Known deviations: none.
