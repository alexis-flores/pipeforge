# Phase 5 report — Co-simulation + bisection

Built `core/cosim`: deterministic stimulus generation with corner cases first
(zeros, ±max, ±1 LSB, ±1.0, sign boundaries; CS-1a); harness generation producing a
plain-port `tb_wrapper.sv` (instantiates `fixedp` + DUT), a cocotb testbench with
valid-signal pacing and valid-gated capture (CS-1b), stimulus/expected JSON from the
bit-exact golden model, and a host-side bit-for-bit comparison with FX-4 error stats
and first-failing-vector reporting (CS-1c, CS-3 machine-readable `CosimResult`).
Verilator/cocotb are probed at call time; absence raises an actionable
`CosimUnavailable` (CS-2/C2). `pipeforge-cli cosim` runs the whole loop headlessly.

Built `core/bisect`: golden intermediates for every node (FX-3) compared against
observed RTL streams; the first divergent stage is reported with node ID, conventional
instance name, cycle, expected/actual values, and whether its inputs matched (BI-1).
Replaying the stage with shifted operand streams distinguishes **wrong-math** from
**delay-skew** ("input X skewed by N cycles — a `PIPE bug"), the dominant real failure
mode (BI-2). `downstream_of_divergence` feeds the timeline's green/red/dimmed
rendering (BI-3 plumbing; the timeline widget already supports it).

Gate evidence: corrupted-stage fixture localized to the correct (first) stage;
delay-skew fixture classified as `delay-skew` with the right operand and N=1; the
CS-4 end-to-end test (`tests/integration/test_cosim_e2e.py`, known-good
sample.m/sample.sv pair against the real nkMatlib RTL) **skips-not-fails** here
(no Verilator; cocotb does not build on Python 3.14) and runs in the CI
`verify-with-tools` job on Python 3.12.

  ruff + mypy --strict(core): clean.
