# Phase 1 report — Core engine

Authored `seed/matlib_audit.py` + `seed/example.m` (the SRS lists these as provided, but they
were absent from the delivery — they were implemented from the §1.5/R2 description and the
nkMatlib README cost model, then frozen). Golden text/JSON audits were captured from the seed
over the five-file fixture corpus (example, normalize3d, rootsqr, feedback, gen500) **before**
refactoring. The seed was then refactored into `core/frontend` (lexer, AST with exact spans,
recursive-descent parser, DAG builder), `core/costmodel` (all latencies derived from
WIDTH/SCALE), and `core/audit` (findings engine: RECIP/CDIV/SERDIV/POW/CSE/FUSE/FEEDBACK;
text/JSON renderers). Golden tests confirm the refactor is behaviorally identical
byte-for-byte modulo the version header, and `pipeforge-cli audit` reproduces the seed CLI
verbatim (Appendix C). Requirements FE-1…5, AU-1…5 are covered by marked tests; hypothesis
property tests pin the parser canonical round-trip; NF-1 (500 statements < 1 s) passes in
~0.05 s.

- Tests: 71 passed. Coverage: core ≈ 90 % (gate: 85 % core), overall 89 %.
  ruff + ruff format + mypy --strict(core): clean.
