# Phase 3 report — GUI shell + Audit view

Built the PyQt6 presentation layer: main window with left icon sidebar (eight
capabilities + Settings), central view stack, status bar (file, WIDTH/SCALE chip,
external-tool dots per Appendix B), collapsible right inspector with span-highlighted
source view, and a collapsible console (UI-1). Workspace-level file/WIDTH/SCALE context
re-audits live and synchronizes selection across views by shared node IDs (UI-2, VZ-2).
The signature **pipeline timeline** (custom-painted cycle ruler; critical path glowing
red, dividers orange) drives both the Audit and Visualizer views (§5.1, VZ-1).
Theming: QSS generated entirely from semantic tokens (TH-1), Gruvbox Dark Soft default
(TH-2), four bundled themes + user-theme directory with validation fallback (TH-3),
live switching with persistence under the platform config dir (TH-4, NF-7), full token
set (TH-5), system-UI/monospace typography with the 11/13/15/20 scale (TH-6).
Keyboard: Ctrl+O / Ctrl+1…9 / Ctrl+R / Ctrl+` (UI-4). Findings table is sortable,
tag-filterable, with click-through to node + source span (UI-6). `core/viz` provides
the layered cycle layout, optional `dot` row refinement, and themed SVG export (VZ-3);
PNG export via widget grab. NF-4 covered by a 60-case fuzz test plus a global
exception hook (log + toast); NF-3 cold-start measured < 2 s in CI.
Screenshots of the Gruvbox Dark Soft default: `docs/screens/01..03-*.png`.

- Tests: 143 passed. Coverage: 89.6 % overall (gate: 80 %). ruff + mypy --strict(core): clean.
- Known deviations: command palette (UI-4, P1) deferred to Phase 8 as the SRS phase plan
  specifies; Golden model/Co-sim/Bisection/Linter/Codegen/DSE views are explicit empty
  states until their phases land.
