# Phase 7 report — Range analysis + design-space exploration

Built `core/ranges`: interval arithmetic with hypothesis-verified containment under
every operator (Phase 7 gate), DAG propagation reporting per node the value range,
required integer bits, overflow risk at the configured WIDTH/SCALE, and
divide-by-near-zero hazards (divisor interval within ±4 LSB of zero) (RP-1); affine
arithmetic (RP-2) preserving linear correlations (`a − a` is exactly 0), selectable
per run and labeled in results; and `recommend_format` (RP-3) deriving LEFT from
propagated ranges (forced even — `usqrt.sv` shifts by LEFT/2, odd LEFT mis-scales
roots) and SCALE from an error budget, then validating empirically via FX-4 runs.

Built `core/dse`: WIDTH/SCALE grid sweeps in parallel worker processes with progress
callbacks and cancellation (DSE-1; cancellation test green), per-point latency /
census / divider count / fixed-vs-float error metrics on a fixed seeded stimulus,
Pareto-front extraction minimizing (error, latency, dividers) with a synthetic-data
correctness test (DSE-2), and disk caching keyed by (source hash, config) plus
CSV/JSON export (DSE-3). The GUI Exploration view runs sweeps off the GUI thread
(UI-3) with progress/cancel, plots all points + the starred Pareto front (pyqtgraph,
theme tokens), and "Adopt selected" updates the workspace WIDTH/SCALE live —
covered by an integration test asserting the status-bar chip reacts.
CLI: `pipeforge-cli ranges` (with `--recommend BUDGET`) and `pipeforge-cli dse`.

- Tests: 220 passed, 2 tool-skipped. Coverage ≈88 % (gate: 80 %).
  ruff + mypy --strict(core): clean.
- Known deviations: vector reductions (sumsqr/norm) use per-element interval bounds
  (vector lengths are not statically known); crossp uses a symmetric product bound.
